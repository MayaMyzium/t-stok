上傳說明（無 data 資料夾版本）
==================================

此資料夾（`combined_website_final`）已將所有預測檔案移到專案根目錄，不再需要建立 `data` 資料夾。請依照以下步驟將網站部署到 GitHub Pages：

1. **解壓縮**：將整個 `combined_website_final` 資料夾解壓縮後，確保包含 `.html`、`.css`、`.js`、`.json`、`.py`、`README.md`、`UPLOAD_INSTRUCTIONS.txt` 等檔案。

2. **上傳檔案**：
   - 登入你的 GitHub 帳號，建立一個新的儲存庫（或使用既有儲存庫）。
   - 點選 `Add file` → `Upload files`，選擇所有檔案（包括 `index.html`、`analysis.html`、`taiwan.html`、`styles.css`、`script.js`、`taiwan.js`、`taiwan_summary.js`、`latest_predictions_taiwan.json`、`realtime_taiwan.json`、Python 腳本等）並上傳至儲存庫根目錄。
   - 如果你使用 Git 客戶端（例如 GitHub Desktop），可將檔案複製到本地儲存庫資料夾後提交 (commit) 並推送 (push)。

3. **提交變更**：輸入 commit 訊息，例如「Add combined crypto and Taiwan stock dashboard」，然後提交。確認所有檔案（包含 JSON、JS、CSS、HTML 以及 `taiwan_summary.js`）均在儲存庫根目錄。

4. **啟用 GitHub Pages**：
   - 前往儲存庫的 **Settings → Pages** 頁面。
   - 在 **Source** 部分選擇 `main` 分支及 `/(root)` 目錄，點擊 Save。
   - GitHub 會自動建置並發布網站。幾分鐘後你可以透過 `https://你的帳號.github.io/儲存庫名稱/` 訪問首頁，並透過 `.../analysis.html` 或 `.../taiwan.html` 存取進一步分析與台股預測頁面。

5. **自訂網域（可選）**：
   - 若要將 Namecheap 的網域指向 GitHub Pages，請在儲存庫根目錄建立 `CNAME` 檔，內容填入你的網域（例如 `example.com`）。
   - 進入 Namecheap 的 **Advanced DNS** 設定，新增四筆 **A 記錄** 指向 GitHub Pages 的 IP（185.199.108.153、185.199.109.153、185.199.110.153、185.199.111.153）以及一筆 **CNAME 記錄**，將 `www` 指向 `你的帳號.github.io`。
   - 回到 GitHub Pages 設定頁確認自訂網域已經填入並啟用 HTTPS。

完成上述步驟後，你的網站將成功部署。若有任何找不到資料的情況，請確認預測 JSON (`latest_predictions_taiwan.json`、`realtime_taiwan.json`) 已放在根目錄，且瀏覽器已正確載入 `taiwan_summary.js`。
